# tech-docs-guard 问题分类(8 类 · finder 检查清单)

每条问题 = 一个 `category` + 三根正交轴(`impact` 严重度 / `type` 缺陷类型 / `conf` 确定度)。
**本轮舍弃 `minor`(瑕疵不查不报),只产 blocker / misleading。** 由原 12 类合并到 8 类(更好处理);括号注原编号。

## A. 可量化(对照代码 / grep 可确定性判 → 可下「确认对不上」、可计数)

- **C1 引用找不到** — 超链接 / 章节锚点失效或点过去是无关内容;文档点名要参考 / 进入的文件 / 目录在仓内不存在。（原 C1 链接 + C2 路径目录）
- **C2 名字 / 接口 / 参数与代码对不上** — 文档写的文件名 / 命令 / 参数名,或函数签名、参数的数据类型表 / schema,与代码真实定义或契约不一致。**多列代码不支持的类型、用错名 = 阻断;漏列代码已支持的 = 瑕疵(本轮不报)。**（原 C3 命名 + C6 接口/schema/参数表）
- **C3 贴出的代码片段本身有错** — 片段语法 / 拼写错,或同段内「声明」与「使用」对不上。**只有标称「粘贴 / 替换进某文件」的【规定式】片段、照做真编不过才升 blocker;【示意式】片段(教写法 / 给例子,读者会自然适配)变量名 / 签名对不上 → 至多 misleading、通常不报**(灵魂③,先 steelman「真会照贴被卡住吗」)。（原 C4）
- **C4 文档自相矛盾** — 同一篇 / 同算子多篇文档对同一事物说法不一致(无需外部代码即可判)。（原 C5）
- **C5 支持范围声明有误** — 产品支持表的 √×、版本 / SOC / 分支适配声明与代码注册不符。**假 √(标支持但代码没注册)= 阻断。**（原 C7）
- **C6 运行结果 / 数值 / 行为说错** — 运行命令后「生成什么 / 在哪 / 应看到什么输出」,或文档里的具体数值 / 公式 / 行为说明,与脚本 / 代码实际不符。（原 C8 命令产物 + C9 数值/公式/行为）

## B. 不可量化(教学判断 → 先 steelman,结论最多「疑似」,绝不写「确认讲错」除非有外部反证)

- **J1 概念讲错 / 术语用混** — 把原理讲反,或用了和框架同名、容易混淆的说法。需外部反证(仓内实现 / 官方文档 / API 语义)才升「确认讲错」,否则「疑似概念风险」。（原 J1）
- **J2 缺步骤 / 没讲「为什么」/ 指代含糊** — 少了必要步骤、没交代前置条件、只给样板没讲关键设计原因、或「改某个文件」却没说是哪一个。（原 J2 没讲为什么 + J3 步骤/前提/指代）

## 三轴 + 判定纪律

- **impact**(开发者影响,按「后果 + 方向」定,不机械按类目):`blocker` 照做会失败(编不过 / 步骤扑空 / 选到用不了的芯片·类型)> `misleading` 困惑但能恢复(死链 / 同篇矛盾 / 示例输出贴错)> `minor` 几乎无后果(**本轮舍弃不报**)。
- **type**(缺陷类型,判定依据决策树):文档里**没有**=`missing`;有但**与代码/数学/自身冲突**=`untrust`;在且对但**表达费解/不一致**=`readable`。
- **conf**:对照代码 / 文档坐实=true 确认 / 线索级=false 疑似。**check**:`code` 须查代码 / `doc` 查文档 / `rule` 查规则。
- **三态(可量化)**:`确认对不上`(强证据反证)/ `疑似`(静态没找到但可能生成/外部/分支)/ `未找到静态证据`(存疑不指控)。
- **三条灵魂**:① 不打玄学分(主观判断先 steelman 再 flag)② grep 不到 ≠ 编造(三态,只有强证据写「确认」)③ **按实际开发者影响判,不按机械字面对照判**(示意 ≠ 规定,见 C3)。
- **互斥落类**:doc→代码名/接口 = C2;doc 片段自身 = C3;doc→doc = C4;同一原文多个问题按最小可修复事实拆条。
